hexo + github 安裝&部署筆記

為何使用hexo

之前空殼的筆記是使用wordpress去建置的，使用約莫兩三年的時間，
發現我最常用的功能只有發佈文章，wordpress想要更改佈景主題又沒有太多時間研究細節T口T
後來發現Hexo建構挺方便的，安裝和部署也不是太困難，這邊就簡單筆記一下。

建置環境

MAC High Sierra > 2020 年使用 Catalina

安裝Node.js

至Node.js官網安裝 node.js
使用node -v可查詢安裝的 node.js 版號

安裝git

主要用於將 Hexo 內容提交到 github 上，這裡是使用 Xcode 已有包含git。

安裝Hexo

npm install -g hexo-cli

初始化Hexo

hexo init blog
blog為自己的資料夾名稱，可替換自己想要的資料夾名稱
注意！hexo在建置過程只能允許空資料夾安裝

安裝相關套件

cd blog
npm install

產生靜態頁面

hexo g

啟動本地伺服器

hexo s
網址輸入http://localhost:4000就可以發現預設會有一篇Hello World的貼文。

yourname請替換成你的帳號

ssh-keygen -t rsa -C "yourmail@gmail.com"

但如果想要自動化部屬 blog，可參考以下資訊。

使用 GitHub Actions 自動部署 Hexo

GitHub Actions 是 GitHub 提供的 CI/CD 服務 ，用於在 GitHub 項目建置、測試、封裝、部署任何專案。

基本上在 yourname.github.io 的內容，是 hexo 透過 hexo g (genarate) 方式產生靜態網頁，
hexo d (deploy) 則將 public 資料夾中的內容推送到 _config.yml 所設定遠端專案的分支中，
並直接完整覆蓋分支下所有內容。
也就是說 yourname.github.io = blog (本地端資料夾名稱) > public 的內容（訪客主要看到blog的網頁）

因此如果透過 hexo 原始碼直接推送到 GitHub 的另一個專案分支上（程式碼專案，非剛剛所說的yourname.github.io 專案），並透過 GitHub 所提供的 GitHub Actions 服務產生靜態網頁檔案，再自動推送到專案分支yourname.github.io

在建置的時候我知道CI/CD服務，但我第一次使用 Github Actions 跌入坑洞，重複建置好幾次XD
簡單說明分支下的內容

Github 分支內容 說明
yourname.github.io Hexo > public 下的內容，靜態網頁
blog_source Hexo 原始碼

把 Hexo 原始碼推送到 GitHub

在 github 新增一個倉庫＋New repository
(與剛剛發佈用不一樣，是專門置放 hexo 的原始檔，並設定為 private)
所以在 github 上會有兩個專案

  • yourname.github.io(放置 hexo 靜態網頁)
  • blog_source (放置 Hexo 原始檔，可隨意取名方便辨認就可)

建立 SSH Key

Github Actions 部署過程中，需要將靜態網頁推送到原先yourname.github.io 專案上，需要建立 SSH Key，提供給 Github Actions 使用。

Code

建立後會在 blog 資料夾底下產生 github-deploy-key 和 github-deploy-key.pub 兩隻檔案（可用編輯器讀取內容）

  • 原始碼專案 (blog_source)
    在 Settings 的 Secrets 去 Add a new secret

    • Name 輸入 HEXO_DEPLOY_PRI
    • Value 請填上私鑰 github-actions-deploy 的內容

  • 靜態網頁專案 (yourname.github.io)
    在 Settings 的 Deploy keys

    • Title 輸入 HEXO_DEPLOY_PUB
    • Key 請填上公鑰 github-actions-deploy.pub 的內容
    • 並勾選 Allow write access

這邊一定要特別注意填寫Name or Title & value 不要填錯專案，不然 deploy 會失敗
